# 微表情渲染增强记录

## 目标

本文件记录 avatar system 中与微表情渲染相关的设计、实现和验证。增强工作从
GiGA 风格模块开始，后续可以在同一入口继续研究皱纹、肌肉联动、时序一致性和
情绪强度控制等方向。

当前原则：

- 保持原有 `GaussianAvatars` 训练与推理路径可用。
- 新增能力必须显式开启，默认行为和既有模型不变。
- 新模块的首次接入采用零残差初始化，未训练的增强模块不改变基础渲染。
- 每次修改后至少进行语法/导入验证；涉及可运行逻辑时增加针对性测试。

## 论文依据

本项目本地论文为 [`pdf/2412.02508v1.pdf`](pdf/2412.02508v1.pdf)，其中第 6 节
提出 Globally-informed Gaussian Avatars (GiGA)。

论文中的核心思路：

- GaussianAvatars 将 Gaussian 绑定到 FLAME 三角面片，基础形变主要受父面片控制。
- 微表情细节可能由非局部区域联动触发，例如眉部变化引起额头纹理变化。
- GiGA 的 Offset-aware Module 从全局 mesh deformation 中提取稀疏关联信息，
  为 Gaussian 的位置、尺度、不透明度、旋转与外观属性预测残差。
- 论文训练流程分为基础 avatar 重建阶段与冻结基础参数、仅优化增强模块的阶段。

## 当前项目接入点

当前渲染链路为：

```text
DEEPTalk motion/FLAME parameters
    -> GSavatar_runs/GaussianAvatars/scene/flame_gaussian_model.py
    -> GSavatar_runs/GaussianAvatars/scene/gaussian_model.py attribute transforms
    -> GSavatar_runs/GaussianAvatars/gaussian_renderer/__init__.py rasterization
```

增强模块放在属性变换层，在进入 rasterizer 前给绑定到 mesh 的 Gaussian 增加
残差。这样不需要修改 CUDA rasterizer，也不会影响 ASR、对话、TTS 或 DEEPTalk
链路。

## v3：属性独立 MLP 与可选高阶 SH

为靠近论文的 Offset-aware Module，当前新增了第一版结构升级：

- 默认仍使用旧的 `shared` head，保证已有 v1/v2 `micro_expression.pth` 能加载。
- 新增 `--micro_expression_head_mode separate`，为 `p/s/alpha/r/H` 分别建立 MLP，
  而不是用一个 shared MLP 输出后再切分。
- 新增 `--micro_expression_appearance_mode sh`，在 `separate` head 下可同时预测
  `features_dc` 和 `features_rest` 残差；默认仍是 `dc`，避免旧 A/B 和旧权重受影响。
- `baseline.mp4` 仍可通过 `--disable_micro_expression` 跳过增强权重，A/B 逻辑不变。

新阶段二实验建议从 v2 的 200k base avatar 开始，输出到新目录，避免覆盖旧结果：

```bash
ROOT=/scratch/e1554543/avatar_system_full
bash "$ROOT/scripts/train_gaussian_subject.sh" nersemble_074_union10 \
  --source "$ROOT/GSavatar_runs/GaussianAvatars/datasets/nersemble_preprocessed/union10_sources/074" \
  --model "$ROOT/data/subjects/nersemble_074_union10/gaussian_micro_attr_200k" \
  --load_model_path "$ROOT/data/subjects/nersemble_074_union10/gaussian_train_200k" \
  --load_iteration -1 \
  --enable_micro_expression \
  --micro_expression_only \
  --micro_expression_head_mode separate \
  --micro_expression_appearance_mode dc \
  --iterations 12000 \
  --interval 2000 \
  --test_iterations 2000 4000 6000 8000 10000 12000 \
  --save_iterations 2000 4000 6000 8000 10000 12000 \
  --checkpoint_iterations 2000 4000 6000 8000 10000 12000
```

若 `dc` 版本稳定，再单独开高阶 SH 实验：

```bash
--micro_expression_head_mode separate --micro_expression_appearance_mode sh
```

## 与原论文 GiGA 的剩余差距

当前项目已经复现了 GiGA 的主干思想：mesh-bound Gaussian、全局 mesh deformation
上下文、Gaussian 属性残差、两阶段训练和阶段二冻结基础参数。但它仍是
`GiGA-style` 工程实现，距离论文设置还有以下差距：

- 全局关系建模仍不同。论文使用可优化的 `W in R^(M x M)` triangle attention
  matrix，`M` 约为改造后 FLAME mesh 的 `10,144` 个 faces，并通过 Sparse-softmax
  在完整 triangle 集合中选择 top-k 关联。当前实现用 query/key 在均匀 anchor
  triangles 上做稀疏查询，不保存完整 face-to-face 关系矩阵。
- Offset-aware Module 仍未完全同构。当前已经支持 per-attribute MLP，但默认仍为
  shared head 以兼容旧权重；位置残差还没有论文中的可学习 `kappa_p`，高阶 SH
  也还需要通过新实验验证稳定性。
- 输入特征并非完全同构。论文使用三角形均值位置、全局旋转矩阵和尺度组成
  `T_i = Concat(T_i, R_i, k_i)`，当前 v2/v3 使用相对中性脸的 deformation features。
- Stage 1 训练预算不足。论文每个 subject Stage 1 训练 `420,000` iterations；
  当前 v2 base 是 `200,000` iterations。
- 数据与评测规模不足。论文使用 9 subjects、多 expression sequences、多视角标准
  评测；当前主要围绕 `nersemble_074_union10` 单 subject 做训练和 A/B 视频。
- CTEG 到 GiGA 的端到端情绪表达链路尚未复现；当前主要使用 DEEPTalk 或已有
  FLAME motion 驱动。
- 还缺 Sparse-softmax/全局机制消融、attention 可视化、区域级微表情误差分析。

## 视频 A/B 对比

训练输出目录中出现 `micro_expression.pth` 后，可以用同一 `point_cloud.ply`、同一
motion、同一音频和镜头渲染 A/B 对照：

```bash
ROOT=/scratch/e1554543/avatar_system_full
bash "$ROOT/scripts/compare_micro_expression_video.sh" \
  "$ROOT/data/subjects/nersemble_074_union10/gaussian_micro_attr_200k/point_cloud/iteration_12000/point_cloud.ply" \
  "$ROOT/outputs/web_recording_1779716215514_20260525_213700/artifacts/flame_motion.npz" \
  "$ROOT/outputs/web_recording_1779716215514_20260525_213700/artifacts/reply.wav" \
  "$ROOT/outputs/micro_expression_compare_nersemble_074_attr"
```

脚本会生成：

```text

enhanced.mp4
side_by_side.mp4
```
